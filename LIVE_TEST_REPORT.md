## 1. Critical Issues Found

### 1.1 Job Submission Takes ~110 Seconds ❌ CRITICAL
**Measured:** Every single job submission (anime, simulation, 3D, story) took **~110,000ms (110 seconds)** to return 202.

**Root cause:** The safety pre-check calls Groq LlamaGuard synchronously in the request handler. LlamaGuard 3 8B (`llama-guard-3-8b`) has been **decommissioned by Groq** and returns a 400 error. The safety service catches this and falls back to keyword-only checking — but the failed API call still takes ~110 seconds to timeout.

**Impact:** Every generation request blocks the user for ~2 minutes before even starting. This is the single most critical bug in the system.

**Fix:**
1. Update LlamaGuard model name in `safety.py` to the current supported model
2. Add a short timeout (5s) to the LlamaGuard API call so failures fail fast
3. Run safety check asynchronously (fire-and-forget for pre-check, or use a background task)

```python
# In safety.py — update model name:
_LLAMAGUARD_MODEL = "meta-llama/llama-guard-4-12b"  # current Groq model
# Add timeout:
groq = AsyncGroq(api_key=..., timeout=5.0)  # fail fast
```

### 1.3 Story Generation — Safety False Positive on "bomb" ❌ HIGH
**Error:** Story about "World War II" failed with `safety_violation: Generated content contains blocked keyword: 'bomb'`

**Root cause:** The keyword blocklist in `safety.py` contains "bomb" as a blocked word. The story plan generated by Groq for WWII naturally includes the word "bomb" (atomic bomb, bombing campaigns, etc.) — this is historically accurate educational content, not harmful.

**Impact:** Any educational topic involving war, chemistry (bomb calorimeter), or physics (nuclear) will fail post-generation safety checks.

**Fix:** Remove overly broad keywords from the blocklist. Keep only truly harmful terms:
```python
# Remove from blocklist: "bomb", "kill", "weapon", "nuclear", "explosive"
# These are legitimate educational terms
# Keep: explicit sexual content, self-harm instructions, etc.
```

### 1.5 Bella TTS — edge-tts Crashes with 500 ❌ HIGH
**Error:** `/api/v1/bella/tts` returns 500. Backend shows no specific error in logs.

**Root cause:** `edge-tts` requires an active internet connection and makes requests to Microsoft Edge TTS servers. The async streaming pattern (`async for chunk in communicate.stream()`) may be failing silently, returning empty bytes, which then triggers the `RuntimeError("edge-tts returned empty audio")`.

**Impact:** Bella has no voice. `tts_available: false` in all chat responses. Lip sync is completely non-functional.

**Free alternatives:**
| Service | Quality | Cost | Notes |
|---------|---------|------|-------|
| **edge-tts** (fix) | High | Free | Fix the async streaming issue |
| **gTTS** (Google TTS) | Medium | Free | Simple, reliable, no key needed |
| **pyttsx3** | Low | Free | Offline, no internet needed |
| **Kokoro via HF** | Very High | Free | Best quality, needs HF token |

**Fix for edge-tts:**
```python
# In bella_service.py — use save-to-file approach instead of streaming
async def _synthesize_speech_with_phonemes(self, text: str):
    communicate = edge_tts.Communicate(text, _EDGE_TTS_VOICE)
    with tempfile.NamedTemporaryFile(suffix='.mp3', delete=False) as f:
        tmp_path = f.name
    await communicate.save(tmp_path)
    with open(tmp_path, 'rb') as f:
        audio_bytes = f.read()
    os.unlink(tmp_path)
    if not audio_bytes:
        raise RuntimeError("edge-tts returned empty audio")
    return audio_bytes, []
```

### Fix 4: model3d_engine.py — Switch to Meshy API
```python
# Add to .env:
# MESHY_API_KEY=your_free_key_from_meshy.ai

_MESHY_API_URL = "https://api.meshy.ai/v2/text-to-3d"

async def _call_meshy_model3d(prompt: str) -> bytes:
    api_key = os.environ.get("MESHY_API_KEY", "")
    headers = {"Authorization": f"Bearer {api_key}"}
    # Step 1: Create task
    r = await client.post(_MESHY_API_URL, json={"mode": "preview", "prompt": prompt}, headers=headers)
    task_id = r.json()["result"]
    # Step 2: Poll until complete
    for _ in range(60):
        await asyncio.sleep(5)
        r = await client.get(f"{_MESHY_API_URL}/{task_id}", headers=headers)
        d = r.json()
        if d["status"] == "SUCCEEDED":
            glb_url = d["model_urls"]["glb"]
            glb_r = await client.get(glb_url)
            return glb_r.content
    raise RuntimeError("Meshy 3D generation timed out")
```
